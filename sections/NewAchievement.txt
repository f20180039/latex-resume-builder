 Problem-Solving & Decision-Making
 Context-Aware Patient Search Component.
The Problem: Doctors needed to enter multiple data
points (Name, Age, Gender, Phone) rapidly, but
standard dropdowns only allowed searching by one
attribute at a time.
The Creative Solution: I engineered a custom search
engine that handles comma-separated string parsing.
I decoupled the UI display from the API query logic,
allowing a doctor to type "John, 30, M" and have the
system intelligently search for the name while
pre-filling the age and gender filters. I used intricate
CSS transforms to ensure the navigation tabs
remained visible and "sticky" on the edge of the
drawer even during complex animations, providing a
seamless UX.
